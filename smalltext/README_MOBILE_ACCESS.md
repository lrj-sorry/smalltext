# 手机如何访问 SmallText 网站

## 方案一：手机和电脑在同一 WiFi 网络下（推荐，简单快捷）

### 步骤 1：确保电脑已启动 HTTP 服务器

1. 确保您已在 `smalltext` 目录下启动 HTTP 服务器：
   ```bash
   python -m http.server 8000 --bind 0.0.0.0
   ```
2. 确认服务器正在运行，显示：`Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...`

### 步骤 2：获取电脑的局域网 IP 地址

#### Windows 系统：
1. 打开命令提示符（Win + R → 输入 `cmd` → 回车）
2. 输入 `ipconfig` 并回车
3. 找到「无线局域网适配器 WLAN」部分
4. 查看「IPv4 地址」，例如：`192.168.166.18`

#### macOS 系统：
1. 打开终端（Launchpad → 其他 → 终端）
2. 输入 `ifconfig en0` 并回车
3. 查看 `inet` 后面的 IP 地址，例如：`192.168.1.100`

#### Linux 系统：
1. 打开终端
2. 输入 `ip a` 并回车
3. 找到无线网卡（通常是 `wlan0` 或类似名称）
4. 查看 `inet` 后面的 IP 地址，例如：`192.168.0.5`

### 步骤 3：手机连接同一 WiFi 网络

确保您的手机已连接到与电脑相同的 WiFi 网络。

### 步骤 4：在手机浏览器访问网站

1. 打开手机浏览器（推荐使用 Chrome 或 Safari）
2. 在地址栏输入：`http://电脑的IPv4地址:8000`
   例如：`http://192.168.166.18:8000`
3. 按下回车键，即可访问 SmallText 网站

## 方案二：手机和电脑不在同一 WiFi 网络下（公网访问）

如果您需要在任何地方都能通过手机访问 SmallText 网站，可以使用内网穿透技术。以下是两种简单易用的方案：

### 推荐方案：使用花生壳（简单易用，有免费版）

#### 步骤 1：下载花生壳客户端

1. 访问 [花生壳官网](https://hsk.oray.com/) 下载并安装花生壳客户端
2. 注册花生壳账号并登录

#### 步骤 2：创建映射

1. 在花生壳客户端中，点击「添加映射」
2. 设置映射信息：
   - 应用名称：SmallText
   - 应用类型：HTTP
   - 内网主机：127.0.0.1
   - 内网端口：8000
3. 点击「确定」，花生壳会生成一个访问地址

#### 步骤 3：手机访问

1. 确保电脑上的 HTTP 服务器正在运行
2. 在手机浏览器中输入花生壳生成的访问地址
3. 按下回车键，即可访问 SmallText 网站

### 备选方案：使用 Cloudflare Tunnel

1. 访问 [Cloudflare Zero Trust](https://dash.cloudflare.com/sign-up/zero-trust) 并注册账号
2. 创建团队并安装 `cloudflared` 客户端
3. 创建隧道并配置路由，将 `http://localhost:8000` 映射到公共域名
4. 在手机浏览器中输入配置的公共域名，即可访问网站

详细步骤请参考 `README_PUBLIC_ACCESS.md` 文件。

## 手机访问注意事项

1. **确保电脑未休眠**：如果电脑进入休眠状态，HTTP 服务器会停止运行，手机将无法访问
2. **防火墙设置**：确保电脑防火墙允许外部访问端口 8000
3. **浏览器兼容性**：推荐使用 Chrome、Safari 或 Edge 浏览器，确保支持 HTML5 和 JavaScript
4. **屏幕适配**：网站已进行移动端适配，可以正常在手机上显示和操作

## 常见问题

### 问题 1：手机无法访问网站

- 检查电脑和手机是否连接同一 WiFi 网络（局域网访问时）
- 检查电脑的 HTTP 服务器是否正在运行
- 检查输入的 IP 地址或域名是否正确
- 检查电脑防火墙是否允许外部访问端口 8000

### 问题 2：访问速度慢

- 确保 WiFi 信号强度良好
- 关闭电脑上的其他占用网络资源的程序
- 考虑升级内网穿透方案到付费版以获得更好的性能

### 问题 3：登录后内容无法加载

- 检查网络连接是否稳定
- 刷新页面重试
- 确保电脑上的用户数据文件（users/userData.js）存在且格式正确
